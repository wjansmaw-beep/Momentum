# Opportunity and Route Composer

Status: Directional, approved for first implementation  
Version: 1.0

## Pipeline

> Live signals -> freshness and sensitivity -> opportunity -> public destination -> time budget -> route request -> Experience Promise -> Capsule

## Opportunity rules

An opportunity is eligible only when:

- its source and retrieval time are known;
- wording can match the evidence;
- the user's available window includes preparation, travel, experience, and return buffer;
- conditions do not contradict the experience;
- a complete Capsule and failure fallback exist;
- the experience remains valuable if the live possibility does not occur.

## Wildlife wording

Use: `reported`, `observed`, `possible`, and `no guarantee`.

Never use: `is here now`, `you will see`, or manufactured scarcity.

## Route plan

The Composer defines:

- transport mode;
- public destination or curated access point;
- outbound, experience, return, and buffer budgets;
- Apple Maps request parameters;
- accessibility and safety unknowns;
- fallback when route calculation or handoff fails.

Apple Maps calculates the navigable route. Momentum may later own scenic loop generation only after trail data, closures, access rights, and route validation are available.

## Memory boundary

Memory stores the experience and optional user media. It does not preserve an exact sensitive wildlife coordinate or imply that the observation was reproduced.
