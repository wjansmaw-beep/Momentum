# ADR-011: The primary opening is a contextual promise

Status: Accepted  
Date: 2026-07-11

## Context

The current prototype opens with the active declaration that extra time has appeared. That flow is valuable, but the approved North Star describes two entrances: a primary contextual suggestion when Momentum opens and a quiet active-intent action underneath it.

## Decision

The primary surface opens with one Experience Promise selected for the current Human Moment.

Below that promise, a visually secondary action lets the user say:

> Ik heb nu extra tijd

That action begins the existing time-first active flow. Both entrances converge at selection, Experience Promise, acceptance and the Experience Capsule. Active intent is not a separate catalogue or competing homepage.

When Momentum lacks enough trustworthy context, the primary surface may show a general permission-light promise, ask one small question or remain quiet. It must not fabricate context merely to fill the opening.

## Consequences

- The `Now` screen implements one locally ranked primary promise and a secondary active-intent entrance.
- Richer media and live context remain future layers; the current suggestion is explicitly labelled as a trial scene.
- “Ik heb tijd nu” remains easy to find but visually secondary.
- The product still presents one main idea rather than a feed.
