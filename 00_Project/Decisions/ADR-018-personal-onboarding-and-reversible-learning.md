# ADR-018 — Personal onboarding and reversible learning

Status: Accepted  
Date: 2026-07-12

## Decision

Momentum starts with a short, permission-free onboarding that asks only for values the user can knowingly choose: name, desired experience directions, usual company, available equipment, acceptable travel time, and preferred initiative level.

The resulting personal model is device-local, visible, editable, and erasable. It may influence selection only through explicit choices and explicit post-experience feedback.

## Learning contract

- `Not now` is situational and never becomes a durable preference.
- Silence, abandonment, and missing data are not negative feedback.
- `Worth it` may gently increase affinity for an experience form.
- `Do again` may additionally mark a specific experience as a favorite.
- `Not for me` is an explicit durable correction and may suppress that specific experience.
- Recent completed experiences receive a temporary repetition penalty so learning does not create a filter bubble.
- The user can inspect and reset all learned signals without deleting memories or onboarding choices.

## Trust boundary

Onboarding does not ask for Calendar, Location, Health, Photos, notifications, messages, or contacts. Each capability is requested progressively only when its immediate value is visible. Proactive behavior remains inactive until separately designed and consented to.

## Consequences

Now, Today, and Discover share the same learning context. Memory becomes an optional positive feedback surface, not an engagement loop. Personalization remains useful without external accounts or cloud profiling, while later integrations must preserve the same correction and deletion controls.
